Latest updates

(2023/02/13)
For the developers, I've applied an MIT license on this project to more clearly state my stance on this project.

(2023/01/29)
For the developers, I've made my HTML more compliant, especially with classes and IDs. I've also reorganized my CSS sheet.

(2022/12/11)
Greatly tweaked the UI and functions for OpenLogger.
  • UI should fit better for users with smaller screens
  • Style for History, Insert, and Settings should look better
  • Added different images for item quantites
  • Changed number font for items
  • History and CSV now display capture time and date based on Gametime (UTC)
    (Note: History logs that do not have a date and time will be assigned the current date and time upon plugin startup)

(2022/09/30)
Fixed an issue regarding incorrect tiers being selected when selecting the radiobuttons for clues during initialization.

Note for people who installed the plugin between April and End of June:
Considering all of the updates that have happened in that timeframe and now, there may be some plugin-breaking side effects, such as incorrect scans or history logs.
I strongly recommend going to Clear Options Menu and selecting Reset OpenLogger to have a fresh experience.
It may be worth exporting your CSV if you haven't done so yet.
